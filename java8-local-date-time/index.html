[Java8 Time API] LocalDate, LocalTime, LocalDateTime 사용법

2017-12-03
Date, Java8, LocalDate, LocalDateTime, LocalTime, Time, 자바8

이번 포스트에서는 Java8 Date/Time API의 LocalDate과 LocalTime 그리고 LocalDateTime 사용법에 대해서 알아보겠습니다. 이 3개의 클래스는 모두 타임존 개념을 가지고 있지 않기 때문에 타임존에 따른 시간 변환이 불가능하다는 공통점을 가지고 있습니다.

LocalDate

LocalDate 클래스는 타임존 개념이 필요없는 날짜 정보를 나타내기 위해서 사용됩니다. 대표적으로 적용이 가능한 사례는 ‘생일’을 떠올릴 수 있을 것 같습니다. 보통 우리가 생일을 얘기할 때는 타임존 개념을 넣지 않기 때문입니다.

위 예제 코드 처럼 LocalDate 클래스는 public 생성자를 제공하지 않기 때문에 객체를 생성할 때는 now()나, of(), parse()와 같은 정적 메소드를 사용하도록 되어 있습니다. 기본 포멧인 yyyy-MM-dd 형태의 문자열을 parse() 메소드에 넘길 수 있습니다.

LocalTime

LocalTime 클래스는 타임존 개념이 필요없는 시간 정보를 나타내기 위해서 사용됩니다. 예를 들어 기상 시간이나 취침 시간과 같은 개인 일정의 경우 대부분의 상황에서는 타임존 개념이 필요하지 않기 때문에 LocalTime을 활용할 수 있습니다.

객체 생성 방법은 LocalDate과 매우 흡사합니다. 기본 포멧은 HH:mm:ss.SSS 이며 plusXxx()나 minusXxx()와 같은 메소드를 통해서 시간이 변경된 새로운 객체를 반환할 수 있습니다. Java8 Date/Time API의 클래스들은 Immutable하기 때문에 한번 생성되면 내부 필드값을 바꿀 수 없기 때문입니다.

LocalDateTime

LocalDateTime 클래스는 타임존 개념이 필요없는 날짜와 시간 정보 모두를 나태나기 위해서 사용됩니다. 즉, 간단하게 LocalDate와 LocalTime을 합쳐놨다고 보시면 됩니다.

객체 생성 방법은 LocalDate, LocalTime과 대동소이하며 기본 포멧은 yyyy-MM-dd'T'HH-mm-ss.SSS 입니다. dateOfBirth2를 생성할 때는 정적 메소드 대신에 Year.of()로 시작하는 Fluent API를 사용하였습니다. 코드 가독성이 좋아지기 때문에 개인적으로 선호하는 방식입니다.
